## 0. The idea (owner's words, mechanized faithfully)

> Improve the momentum-pullback-continuation strategy by (1) auto-computing dynamic **Volume Profile levels** and **shielding the stop-loss behind strong VP levels that won't break down easily**; (2) using **order flow, actively calculated**, as real confluence for *where* to place the SL and *when* to cut a trade: the EA must calculate the order flow required to break the VP safe zone the stop hides behind, detect that change instantly, and **cut the trade when a calculated 65% violation risk is reached**. This is **not a change of strategy — it is arming the EA constantly with confluence data that decides its behavior**. Prediction: backtesting this addition increases the edge and expands the tradable universe. Also: an **active dashboard displaying the bot's scans and decisions** — the EA's thought process during scanning and trade management.

Mechanization decisions (expert, binding):
- "Strong VP level" → high-volume node (HVN) / point-of-control (POC) / value-area edge from a **rolling M1 tick-volume histogram** (window N pre-registered per grid below), each level carrying a **strength score** (relative volume mass, age, retest count).
- "Order flow" → the only order flow that exists on retail MT5 CFD feeds: **tick-volume proxies** — candle-signed delta, tick-rule delta, CVD slope, buyer pressure share, delta acceleration (OrderFlowLite semantics, measured IC on gold ≈ +0.02..0.06: a lean, not a law).
- "Calculated 65% violation risk" → a **walk-forward-calibrated probability** P(level violated within h bars | OF features, level strength), calibrated so that "65%" empirically means 65% (isotonic/logistic; Brier + reliability curves reported). 65 is the pre-registered action threshold; 55/75 are sensitivity cells. An uncalibrated score crossing an arbitrary 0.65 is NOT acceptable.
- "Not a change of strategy" → entry engine untouched. But SL placement and early exit ARE the exit engine — the most heavily validated part of this system — so they enter live only through the full gate, staged shadow-first.

## 3. Protocol (pre-registered; stages are gates, not suggestions)

**S0 — Feature layer (offline Python; no EA change).**
Data audit first (M1 depth per venue/symbol; fallback = M15 range-distributed volume, marked lower-fidelity). `backtest/vp_engine.py`: rolling VP histogram → POC/HVN/LVN/VA-edges + strength ∈ [0,1]. `backtest/of_features.py`: signed delta, CVD slope, pressure share, delta acceleration per M15 bar. Parity: Python features vs MQL5 export on identical bars, maxdiff reported.

**S1 — Violation model.** Event: price CLOSES through a level by ≥ k×ATR within h bars of first touch. Grid (pre-registered, total cells counted into DSR trials): VP window N ∈ {1d, 3d}; k ∈ {0.15, 0.30}; h ∈ {4, 8}. Model: logistic on OF features + strength, walk-forward refit monthly, isotonic calibration. Outputs: AUC, Brier, reliability curve.

**S1.5 — KILL GATE (before any backtest spend).** The model must beat ALL of:
(a) **placebo levels** — random prices matching the real levels' distance-from-price distribution;
(b) **stale levels** — yesterday's VP applied to today;
(c) **shuffled OF** — features permuted within-day (marginals preserved, timing destroyed).
Level-strength informativeness gates the SHIELD arm; OF incremental lift gates the CUT arm. An arm that fails dies here. **Accept:** real ≥ placebo+stale+shuffled on AUC AND Brier, out-of-fold, per gate arm.

**S2 — Counterfactual backtest** (only for surviving arms; harness = `scalper_confluence.simulate_symbol_c` + `walkforward_dsr`, 12 spread-gated real-cost Deriv M15 symbols):
- Arm A (shield): SL snapped behind the nearest strong adverse-side level within **0.8–1.3 ATR**; risk-sized to actual distance (1R = real cash risk); TP stays fixed 3.0 ATR (pre-registered); costs as fraction of actual R; lot-step quantization modeled.
- Arm B (cut): market-close when calibrated P(violation of the adverse-side shield) ≥ 65 (55/75 sensitivity), only while a shield level exists between price and SL.
- Arm A+B combined (the owner's actual claim).
- **Control 1:** v1.27 with SL mechanically widened to Arm A's mean distance (separates "shielding" from "wider stops stop out less").
- **Control 2:** random-cut placebo — same cut fraction, random timing (separates model information from mechanical truncation).
- Cut decisiveness: per-cut 3-cell transition matrix vs the bracket (saved-from-SL / truncated-winner / would-have-timed-out); cut-EV = P(save)·loss-avoided − P(truncate)·win-foregone − exit spread; must be **positive and beat Control 2**. If cuts don't concentrate in the saved-from-SL cell, it is the same dead truncation class — reject fast.
- Full house gate: stitched-OOS quarters, paired per-signal t, walk-forward, **DSR ≥ 0.95 at the incremented cumulative trial count (68 + all cells including placebos)**, 2× cost stress, no sign-flip across symbols/quarters.
- **S2b (only on PASS):** universe-expansion test on the 29-instrument diverse set + FTMO cost survivors — the "more assets" prediction, tested not assumed.

**S3 — Shadow mode (live, zero behavior change).** v1.29+: EA computes VP/OF live, LOGS would-be decisions (shield level, would-move-SL-to, P(violation), would-cut) to the decision CSV and panel; `InpVpofLive=false` hard default. ≥ 2 weeks; agreement metric vs S2 expectations.

**S4 — Live enable**: input flag flip, only after S2 PASS + S3 agreement + explicit owner sign-off.
